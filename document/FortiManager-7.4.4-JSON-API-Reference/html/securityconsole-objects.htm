import/dev/objs (command)

Import objects from device to ADOM, or from ADOM to Global.

Supported methods: exec

Object URL:
Command object/securityconsole/import/dev/objs
Attribute ListDescriptions
add_mappingsoption
Automatically add required dynamic mappings for the device during the search stages.
When used in policy_search action, add dynamic interface and zone mappings; in obj_search action, add ADOM object mappings.
This attribute is not available for the do action.
"disable"[Default value]
"enable"
adomdatasource, refer to adom object (maximum 1) (36 bytes)
Source ADOM name.
dst_namestring (36 bytes)
Name of the policy package where the objects are to be imported. If the package does not already exist in the database, a new one will be created.
dst_parentdatasource, refer to policy package object (maximum 1)
Path to the folder for the target package. If the package is to be placed in root, leave this field blank.
if_all_objsoption
"none"[Default value]
"all"
"filter"
if_all_policyoption
"disable"[Default value]
"enable"
import_actionoption
"do"[Default value]
Perform the policy and object import.
"policy_search"Preprocess and scan through device database to gather information about policies that need to be imported. Can automatically add interface and zone mappings if add_mappings is enabled.
"obj_search"Preprocess and scan through device database to collect objects that are required to be imported. Can automatically add object mappings if add_mappings is enabled.
namedatasource, refer to device object (maximum 1) (36 bytes)
Source device name.
positionoption
"bottom"
"top"[Default value]
vdomdatasource, refer to device vdom object (maximum 1) (36 bytes)
Response DataDescriptions
taskdatasource, refer to task (maximum 1)
