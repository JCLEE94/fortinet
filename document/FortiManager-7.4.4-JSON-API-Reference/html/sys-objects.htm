login/challenge (command)

Answer a log in challenge question, used following a login/user or login/challenge command.

Supported methods: exec

Object URL:
Command object/sys/login/challenge
Attribute ListDescriptions
answerstring (1024 bytes)
Answer to the challenge question from previous request.
sessionstring
Session ID returned from login/user or previous login/challenge command.
Response DataDescriptions
challengestring (1024 bytes)
New challenge question that must be answered with another login/challenge command. If no further question is returned, then the log in process is completed.
sessionstring
If there is further challenge question(s), the session is used to along with the answer for the next login/challenge command.

 

login/user (command)

Log into the device with user name and password. A new session cookies will be generated and returned as a property of the response object.

Supported methods: exec

Object URL:
Command object/sys/login/user
Attribute ListDescriptions
passwdstring (128 bytes)
userstring (128 bytes)
Response DataDescriptions
challengestring (1024 bytes)
The log in challenge question. User must follow this request with a login/challenge command to complete the log in process. If no log in challenge is used, the log in process is completed.
sessionstring
If there is a log in challenge, a temporary session is created. Use this session along with the answer to complete the log in process with the login/challenge command.

proxy/json (command)

Send and receive JSON request to/from managed devices. The response will be an array of data, one for each queried device.

Supported methods: exec

Object URL:
Command object/sys/proxy/json
Attribute ListDescriptions
actionoption
Specify HTTP action for the request.
"get"
"post"
"put"
"delete"
payloadobject
JSON payload of the request. The payload will be URL-encoded and becomes the "json" field in the query string for both GET and POST request.
resourcestring
URL on the remote device to be accessed.
targetdatasource, refer to device object, or group object
timeoutunsigned int32
Maximum time in seconds to wait for response from remote device. If no response recieved from remote device before timeout expires, the proxy will return with error. If the proxy request is sent to multiple devices, it will not affect the response for other devices that already returned a result. Valid range: 10 to 28800.
Default value: 60
Response DataDescriptions
responseobject
If success, the data retrieved for the device.
statusobject, refer to result object
The error code and message returned for the device.
targetstring
The name of the device where the set of data is belonged to.
