Common Objects


Scope object

An array of objects used for setting or retrieving object members of a parent object. 
	"object member": [
		{
			"name": "...",
			"vdom": "..."
		},
		{
			"name": "..."
		},
		{
			"name": "...",
			"is group": ...
		},
		  ...
	]
When referencing to a device, the object should contain both "name" and "vdom" (unless otherwise noted).
When referencing to other objects (ie. device groups), only the name attribute is used.

For some cases where scope is supported for both device and device group and where device VDOM name is optional, additional attribute "is group" can be specified to force the scope member to be interpreted as device group. Otherwise, the scope member is interpreted as device by default.


Filter object

An array of strings and/or integers that removes unwanted matches from the result. Only results that match the filter conditions will be returned. 
	"filter": [ <source>, <operator>, <target1>, <target2>, ... ]
Unless otherwise specifid, the source for the filter can be an attribute or meta attribute of the object, or an external attribute of a referenced object; the target can be a string or numerical value. The operator can be one of the following:

Operator# of target(s)Descriptions
"=="1Equal to
"!="1Not equal to
"<"1Less than
"<="1Less than or equal to
">"1Greater than
">="1Greater than or equal to
"&"1Bitwise AND, target can be integer value only, test if (source & target) = 0
"&"2Bitwise AND, target can be integer value only, test if (source & target1) = target2
"in"1 or moreTest if source is one of the values in target
"contain"1If source have a list of values, test if it contains target
"like"1SQL pattern matching, where target is a string using % (any character) and _ (single character) wildcard
"!like"1Not like, inverse of "like" operation
"glob"1Case-sensitive pattern matching with target string using UNIX wildcards
"!glob"1Not glob, inverse of "glob" operation
"&&"1Logical AND operator for nested filter with multiple criteria, where source and target must be another filter
"||"1Logical OR operator for nested filter with multiple criteria, where source and target must be another filter
For inverse matches, an "!" element should be appended at the beginning of the array, before the source.


Sorting object

An array of objects to specify the order of matching entries in the result data array. 
	"sortings": [
		{ "attribute": order },
		{ "attribute": order },
		  ...
	]
The attribute specifies the name of the attribute to be sorted. This can be point to an external attribute of a referenced object. Multiple attributes can be specified if the data is to be sorted with 2 or more levels.

Order can be either 1 for ascending, or -1 for descending.


Expand member object

An array of objects to specify the object types and attributes of object member to be returned along with the fetch. The result from this fetch will be returned as a "expand member" sub-object in the result data object. 
	"expand member": [
		{
			"url": "...",
			"fields": [ ... ],
			"meta fields": [ ... ]
		},
		{
			"url": "...",
			"fields": [ ... ],
			"meta fields": [ ... ]
		},
		  ...
	]
The url attribute specifies the type of the member object to be fetched. This is done by referencing to table URL without module prefix (ie. "/device" for "dvmdb/device"). Any object member whose URL is not included in the expand member array will not be returned in the result.

Similar to the parameter in get method, fields and meta fields arrays limit the output result. The expand member sub-object in result will contain only attributes and meta attributes specified in the array. However, if field is no specified, entire member object will be returned.


Section object

An object used to modify section attributes. 
	"data": {
		"attr": "...",
		"name": "..."
	}
The attr attribute can be either "global-label" for section in sequence, or "label" for section in interface-pair view. If not specified, "global-label" will be used.

Section title is specified in the name attribute for set request.
