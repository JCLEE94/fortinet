System Template Methods


Get method (table)

Retrieve all objects from a table that matches the criteria.

Request:
{
   "method": "get",
   "params": [ {
      "url": "...",
      "option": "...",
      "attr": "...",
      "loadsub": ...,
      "get used": ...,
      "filter": [ "src", "op", "target", ... ],
      "fields": [ ... ],
      "sortings": [ { "field", 1 }, ... ],
      "range": [ a, n ]
   } ],
   "session": "...",
   "id": 1
}
Result:
{
   "method": "get",
   "result": {
      "url": "...",
      "data": [ { ... } ],
      "status": { "code": 0, "message": "..." }
   },
   "id": 1
}

 

Parameter and Returned ValuesDescriptions
urlstring
Path to the table, refer to the object list. Do not specify object ID in the URL.
option(optional) string
Set fetch option for the request. If no option is specified, by default the attributes of the objects will be returned.
"count"Return the number of matching entries instead of the actual entry data.
"scope member"Return a list of scope members along with other attributes.
"datasrc"Return all objects that can be referenced by an attribute. Require attr parameter.
"get reserved"Also return reserved objects in the result.
"syntax"Return the attribute syntax of a table or an object, instead of the actual entry data. All filter parameters will be ignored.
attr(optional) string
The name of the attribute to retrieve its datasource. Only used with datasrc option.
loadsub(optional) int16
Enable or disable the return of any sub-objects. If not specified, the default is to return all sub-objects.
get used(optional) int16
filter(optional) filter object
Filter the result according to a set of criteria.
fields(optional) array
Limit the output by returning only the attributes specified in the string array. If none specified, all attributes will be returned.
sortings(optional) sorting object
Specify the sorting of the returned result.
range(optional) range
Limit the number of output. For a range of [a, n], the output will contain n elements, starting from the ath matching result.
data(optional) table
Upon success, return an array of objects that matches the criteria. If the count option is used, an integer value is returned instead.
statusresult object
Error code and message for the command. Refer to the error code list for all possible return values.

 

Get method (object)

Retrieve an object or table object from the database.

Request:
{
   "method": "get",
   "params": [ {
      "url": "...",
      "option": "..."
   } ],
   "session": "...",
   "id": 1
}
Result:
{
   "method": "get",
   "result": {
      "url": "...",
      "data": { ... },
      "status": { "code": 0, "message": "..." }
   },
   "id": 1
}

 

Parameter and Returned ValuesDescriptions
urlstring
Path to the object or table object, refer to the object list.
option(optional) string
Set fetch option for the request. If no option is specified, by default the attributes of the object will be returned.
"scope member"Return a list of scope members along with other attributes.
"chksum"Return the check-sum value instead of attributes.
"datasrc"Return all objects that can be referenced by an attribute. Require attr parameter.
data(optional) object
Upon success, return the object retrieved by the command. If the chksum option is used, an integer value is returned instead.
statusresult object
Error code and message for the command. Refer to the error code list for all possible return values.

      "url": "...",

   "session": "...",

      "url": "...",
      "data": { ... },
      "status": { "code": 0, "message": "..." }

Parameter and Returned ValuesDescriptions

Path to the table or the new object, refer to the object list.

Clone method

Create a duplicate entry of an object in a table, with some or all of its non-key attributes.

Request:
{
   "method": "clone",
   "params": [ {
      "url": "...",
      "data": { ... }
   } ],
   "session": "...",
   "id": 1
}
Result:
{
   "method": "clone",
   "result": {
      "url": "...",
      "data": { ... },
      "status": { "code": 0, "message": "..." }
   },
   "id": 1
}

 

Parameter and Returned ValuesDescriptions
urlstring
Path to the source object entry, refer to the object list. This must point to a single object in a table, which the key for the source object must be included in the URL.
data(optional) object
In Request:
Contains a list of attributes and values that need to be changed on the cloned object. All other attributes, except for the key, will be copied from the source object.
In Response:
The new key of the cloned object.
statusresult object
Error code and message for the command. Refer to the error code list for all possible return values.
