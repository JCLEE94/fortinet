Configuration Database (For v6.4)


This documentation applies to v6.4 devices or ADOM, latest release (build 2087).

Information about other release version can be found in the following links:
Version 7.4 latest release, build 2690
Version 7.4 older release, build 2462
Version 7.2 latest release, build 1681
Version 7.0 latest release, build 623
Version 6.4 older release, build 1716
Version 6.2 latest release, build 1263


This module is responsible for the configuration of managed devices. FortiManager has multiple configuration sets:
  • Global (/pm/config/global/) scope is all ADOMs, and configurations are copied to ADOM during global package assignment.
  • ADOM (/pm/config/adom/) configurations are managed by packages and templates. Each template and package have their scope restricted to the a subset of the ADOM's devices that are assigned to the package, and configurations are copied down to device during install.
  • Device (/pm/config/device/global/, /pm/config/device/vdom/) scope is one device only.

Whenever configuration for Global or ADOM is modified, it will eventually be copied to Device config database after next installation. A few selected device-level objects are listed in this documentation. For other objects that are not supported by higher-level configuration sets, JSON requests to directly access and modify the configuration at the Device level can be done by a linear translation of FGT CLI object paths. Depending on OS version and platform type, the availability of objects and attributes will be different on different devices.

An example at the end of this section in documentation shows how FGT CLI translation works.
